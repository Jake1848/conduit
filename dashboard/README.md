# Conduit Console

The operator dashboard for **Conduit** — Bitcoin/Lightning payment infrastructure for
autonomous AI agents. Monitor a fleet of agent wallets, watch live machine-to-machine
settlements, set spending policy, audit every transaction, and manage API keys.

Built with **Next.js 14 (App Router)** · **TypeScript** · **Tailwind CSS** · **Chart.js v4** ·
**Lucide**. It talks **directly** to the Conduit API from the browser using an operator API
key — there is no backend/BFF in this app.

## Quick start

```bash
cd dashboard
npm install
cp .env.example .env.local      # already present; points at the regtest sandbox
npm run dev                     # http://localhost:3000
```

On first load you'll see a **Connect** screen — paste an API key. For the regtest sandbox
use **`ck_test_regtest_root_key`** (admin scope). The key is stored in `localStorage` and
sent directly to the API on every request.

### Reaching the regtest sandbox (rich dev data)

The regtest API runs on the Conduit host and is bound to localhost. Open an SSH tunnel so
`http://localhost:8002` reaches it from your machine:

```bash
ssh -i ~/.ssh/conduit_key -L 8002:localhost:8002 root@167.233.27.130 -N
```

It has **200+ agents** and **1000+ transactions** continuously generated by an automated
test runner, so every page is populated and the live feed actually moves.

### Pointing at testnet (public)

```bash
# .env.local
NEXT_PUBLIC_API_URL=https://api.conduit.energy
```

> **CORS:** the dashboard calls the API directly, so the API's `ALLOWED_ORIGINS` must
> include this app's origin. The regtest API already allows `http://localhost:3000`. For a
> deployed build, add the deployment's origin (e.g. your Vercel URL) to the **testnet**
> API's `ALLOWED_ORIGINS`.

## Auth

There are no user accounts — the API key **is** the auth:

1. Connect screen accepts an API key → stored in `localStorage`.
2. Every request sends `Authorization: Bearer <key>`.
3. Scope is detected by probing `GET /v1/api-keys` (admin) → falling back to `GET /v1/agents`
   (member). Admin keys see the **API Keys** view; member keys don't.
4. The sidebar footer shows the masked key + tier + network, with a **Disconnect** action.

## Pages

| Route | View | Real API usage |
|---|---|---|
| `/` | **Overview** | Treasury = Σ agent balances; active count; live tx/min + avg settlement + charts from polled transactions. |
| `/wallets` | **Wallets** | `GET /v1/agents` + per-page balances & tx counts; search/filter; bulk create; CSV export. |
| `/wallets/[id]` | **Agent Detail** | Balance, credit/debit, policy editor (`GET`/`POST /policy`), tx history, `POST /v1/invoices`. |
| `/audit` | **Audit Log** | Transactions aggregated across agents, client-side filtered, CSV export. |
| `/keys` | **API Keys** | `GET`/`POST`/`DELETE /v1/api-keys`; secret shown once in a modal. Admin only. |
| `/policies` `/network` `/webhooks` `/sandbox` `/docs` | Placeholders | Styled "coming soon". |

### Notes on real-data deltas from the design mock

- The API's agent objects carry **no balance** and **no scope** — balances are fetched per
  agent (`/balance`); "scope" is derived from the agent name's role prefix.
- There is no metrics or websocket endpoint, so the Overview metrics + live feed are
  **computed from polled transactions** across a sample of active agents (polled every ~4s);
  the feed prepends genuinely new transactions with the design's enter animation.
- `{id}` routes use the canonical `agt_…` id (the agent **name** is not accepted), so the
  detail route is `/wallets/agt_…`.
- The network pill reflects the **real** network from `/v1/health` (regtest/testnet/mainnet)
  rather than a hardcoded "MAINNET".

## Design system

Tokens (colors, type, spacing, radii, shadows) from the handoff are configured in
`tailwind.config.ts`. The component CSS in `app/globals.css` is ported from the handoff's
`reference/src/ds.css` for pixel-fidelity and wired to `next/font` (Inter + JetBrains Mono).

## Deploy to Vercel

```bash
# from the dashboard/ directory
vercel
```

Set `NEXT_PUBLIC_API_URL` in the Vercel project (Production → `https://api.conduit.energy`),
and add the Vercel domain to that API's `ALLOWED_ORIGINS`. Build command `next build`,
output handled by Vercel's Next.js preset. Root directory: `dashboard`.
